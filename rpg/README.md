# ⚔️ 开放世界 RPG (Google Apps Script + Three.js)

一款跑在 **Google Apps Script Web App** 里的第一人称 3D 开放世界 RPG 原型。

> 新手村出生 → 探索野外打怪 → 爆装备 → 回熔炉合成更强装备 → 挑战更强的怪 → 升级解锁新区域

3D 渲染由浏览器端的 **Three.js** 完成，GAS 仅负责托管网页（和你的扫雷项目同一套架构）。存档存在浏览器 `localStorage`，认设备。

---

## 🎮 玩法

| 系统 | 说明 |
|------|------|
| **区域** | 新手村（安全/回血/熔炉）· 翠绿平原 · 幽暗森林 · 枯骨荒原 · 王国之城（安全/熔炉） |
| **战斗** | 屏幕中心准星瞄准怪物，左键攻击；有暴击（×2）；怪物会追击并反击 |
| **爆装备** | 击杀怪物按概率掉落武器/护甲，6 档稀有度：普通→精良→稀有→史诗→传说→神话 |
| **熔炉合成** | 靠近熔炉按 E/F，放入 2 件**同类型**装备，花金币熔炼出更高等级、可能升阶稀有度的装备 |
| **成长** | 击杀获得经验/金币，升级提升攻防和生命上限；等级达标解锁更高级区域 |
| **死亡** | HP 归零损失部分金币，在新手村复活 |

### 操作
- **WASD** 移动 · **Shift** 疾跑 · **空格** 跳跃
- **鼠标** 转视角（若浏览器锁定鼠标）；**按住左键拖动** 转视角（后备模式）
- **左键** 攻击 · **E** 交互/熔炉 · **F** 熔炉 · **I** 背包 · **ESC** 释放鼠标

## 📁 文件结构（GAS 项目）

| 文件 | 作用 |
|------|------|
| `Code.gs` | `doGet()` 返回 Index 模板；`include()` 内联 CSS/JS |
| `Index.html` | 页面骨架 + 所有 UI 容器；用 `<?!= include('CSS') ?>` / `include('JS')` 内联 |
| `CSS.html` | 全部样式（HUD、背包、熔炉、稀有度配色等） |
| `JS.html` | 游戏引擎全部逻辑（Three.js 场景、控制、怪物、战斗、装备、熔炉、存档） |

## 🚀 部署到 Google Apps Script

1. 打开 [script.google.com](https://script.google.com) → **新建项目**。
2. 按上表创建 4 个文件，文件名必须完全一致：
   - `Code.gs`（默认已有，粘贴内容）
   - `Index.html`、`CSS.html`、`JS.html`（**新建 → HTML 文件**，注意名字不要带 `.html` 后缀，GAS 会自动补）
3. 分别把本仓库 `rpg/` 下对应文件内容粘贴进去，保存。
4. 右上角 **部署 → 新建部署 → 类型选“网页应用”**。
   - 执行身份：我
   - 谁可以访问：任何人
5. 复制生成的网页应用 URL，用浏览器打开即可游玩。

> ⚠️ **关于鼠标锁定**：GAS 网页应用运行在 Google 的跨域 iframe 中，浏览器的 Pointer Lock（无限鼠标转视角）通常会被禁用。本游戏已内置**后备方案**：检测到无法锁定时，自动切换为“**按住左键拖动转视角、单击攻击**”，保证可玩。

## ⚠️ 为什么这是「单机」而不是真·MMO？

GAS 不是为实时游戏服务器设计的（无 WebSocket、单次执行 6 分钟超时、有每日配额），无法支撑「多人实时同屏」的 MMO。本项目是**第一人称单机 RPG**，包含完整的核心循环，是通往真正 MMO 的第一步。

若日后要做真正的多人在线，需要更换后端技术栈：
- 前端：Three.js / Babylon.js（可复用本项目的渲染与玩法逻辑）
- 后端：Node.js + WebSocket（Socket.io）做实时同步
- 数据库：MongoDB / PostgreSQL
- 托管：Railway / Fly.io / 云服务器
